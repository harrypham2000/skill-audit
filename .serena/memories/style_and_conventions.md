Code is small, strict TypeScript with ES modules and semicolon-heavy formatting. Naming uses camelCase for functions/variables and SCREAMING_SNAKE_CASE for rule tables. Files are focused by responsibility (`discover.ts`, `audit.ts`, `deps.ts`, `scoring.ts`, `reporter.ts`, `types.ts`). Types are defined as interfaces in `src/types.ts`; however, some implementation files still use `any` and need tightening. Error handling currently prefers soft-fail/ignore patterns over throwing.
